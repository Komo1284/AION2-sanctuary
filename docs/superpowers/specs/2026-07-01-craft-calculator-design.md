# 응룡왕 악세서리 제작 최저비용 계산기 설계

작성일: 2026-07-01
대상 파일: `craft.php` (비밀번호 없이 접근하는 독립 페이지)

## 1. 목적

아이온2 응룡왕 악세서리(목걸이/귀걸이/반지)를 제작할 때, 어떤 루트가 가장 저렴한지
계산해 비교해주는 웹 도구. 사용자는 현재 보유 아이템과 재료 시세를 입력하고, 응룡왕까지
도달하는 데 필요한 총재화(키나 포함)를 루트별로 확인한다.

- 최종 목표 아이템: **응룡왕(기본)**. 빛나는 응룡왕은 옵션으로 키나 비용만 추가 표기.
- 대상 악세서리 3종: 목걸이(class2=303), 귀걸이(304), 반지(305). 추후 확장 가능.

## 2. 도메인 (inven 스크레이핑 기준)

데이터 출처: `https://aion2.inven.co.kr/db/craft/?race=1,0,3&class2=303` 등. JS 렌더링이라
browse 도구로 스크레이핑.

### 제작 티어 (하위 → 상위)

저티어(루비→장인→달인)는 **계산에서 생략**한다. 이유: 결정사항에 따라 중간 아이템
(`달인의 빛나는 ○○ 목걸이` 등)은 시장 완제품 가격으로만 입력받고 하위 제작반을 전개하지
않기 때문. 즉 `달인의 빛나는` 아이템은 가격이 매겨진 잎(leaf) 재료로 취급한다.

용왕 라인: 진룡왕 → 백룡왕 → 명룡왕 → 천룡왕 → 현룡왕 → 응룡왕.
각 티어는 `기본`과 `빛나는` 두 버전을 가진다.

- `빛나는 X` = `X` + 키나 (승급 레시피)
- 각 제작 시 **COMBO 25%** 확률로 빛나는 버전이 공짜로 추가 산출됨

### 레시피 종류

- **계승(inherit)**: `빛나는 하위티어 ×1` + `제작 계승석: 장신구 ×1` + `해당티어 코어 ×1` + 재료
- **코어직접(direct-core)**: `해당티어 코어 ×N` + 재료
- **달인빛나는직접(direct-master)**: `달인의 빛나는 ○○ 목걸이 ×N` + 재료
- **빛나는승급(shine)**: `기본 ×1` + 키나

진룡왕은 계승 레시피가 없다(코어직접/달인빛나는직접만).

### 응룡왕 (유동적 — 오늘 업데이트 반영 대기)

2026-07-01 현재 inven에는 응룡왕이 **달인빛나는직접(달인의 빛나는×5)** 한 가지만 존재.
단, **오늘 업데이트로 `빛나는 현룡왕 → 응룡왕` 계승 레시피가 추가 예정**이다. 사용자가
23:00 서버오픈 후 정확한 레시피를 제공할 예정. 그 전까지는 **임시 추정치**로 응룡왕 계승
레시피(빛나는 현룡왕×1 + 계승석×1 + 응룡왕 코어×1 + 추정 재료)를 넣고 `is_estimated=1`로
표시하며, 관리자가 나중에 정확한 값으로 수정한다.

## 3. 비용 모델

모든 아이템/재료를 노드로 본다.

```
effective_cost(노드) = min(
    입력된 시장가 (있으면),
    각 레시피 r 에 대해:  Σ_{재료 m in r} effective_cost(m) × qty(m,r) + kina(r)
)
```

- **코어** = `effective_cost = 0` (시즌 무료 아이템)
- **키나** = kina_cost 로 총비용에 포함
- **달인의 빛나는 / 계승석 / 각종 제련석·비늘·분노 시리즈·원석·오드** = 사용자 입력 시장가(잎 재료)
- **보유 아이템 선택 시**: 그 아이템의 effective_cost를 0으로 고정하고 응룡왕까지 재계산
- 메모이제이션으로 재귀 계산. 그래프는 작아서(≈6티어 × ≤3레시피) 성능 문제 없음

이 구조 하나로 계승/코어/직접 루트가 자동 비교된다. 억지 루트 하드코딩 없음.

### COMBO 처리 (둘 다 표시)

- **확정비용**: COMBO 무시, 레시피 재료를 그대로 1회분 비용으로 계산
- **기대값비용**: 제작 시 25% 확률로 빛나는 버전을 공짜 획득 → 다음 계승에 필요한 빛나는
  아이템/키나를 0.25 기대값만큼 절감해서 계산

두 값을 나란히 보여준다.

## 4. 계산 & 루트 비교

1. 응룡왕 effective_cost(확정/기대값) 계산 + 선택된 최저 경로 트리 복원
2. **주요 루트 비교표** — 각 루트의 총재화를 계산해 정렬, 최저가 강조:
   - 전체 계승 루트 (하위부터 계승 체인)
   - 코어 활용 루트 (코어로 현룡왕/천룡왕까지 직접제작 후 계승)
   - 응룡왕 직접제작 루트 (달인의 빛나는×5)
3. 루트별 **재료 소모 breakdown** (재료명 × 수량 × 단가 = 소계)
4. 보유 아이템이 있으면 거기서부터의 잔여 비용만 표시

## 5. 데이터 모델 (MySQL, 기존 앱 패턴)

```sql
craft_materials (
  id, name UNIQUE, unit_price INT DEFAULT 0,
  is_core TINYINT DEFAULT 0,       -- 1이면 항상 0원
  category VARCHAR,                 -- 제련석/비늘/분노/원석/오드/중간아이템/계승석/키나 등
  updated_at DATETIME, updated_ip VARCHAR
)
craft_recipes (
  id, accessory ENUM(목걸이,귀걸이,반지), output_name,
  recipe_type ENUM(계승,코어직접,달인빛나는직접,빛나는승급),
  tier INT, kina_cost BIGINT DEFAULT 0, combo_rate DECIMAL DEFAULT 0.25,
  is_estimated TINYINT DEFAULT 0, note VARCHAR
)
craft_recipe_inputs (
  id, recipe_id, material_name, qty INT
)
```

앱 시작 시 테이블 없으면 자동 생성 + inven 데이터로 seed (기존 `index.php`의
`CREATE TABLE IF NOT EXISTS` + 마이그레이션 패턴 따름).

## 6. UI 흐름 (`craft.php`, 비번 없이 접근)

- 상단: 악세서리 선택(목걸이/귀걸이/반지) + 현재 보유/장착 아이템 선택(없음 / 진룡왕~현룡왕)
- 중앙: 루트별 비용 비교 카드(최저가 강조) + 각 루트 재료 소모 breakdown, 확정/기대값 병기
- 재료 시세 편집 표: 재료명 · 현재가 · 최종 갱신 시각 · 인라인 수정. **누구나 저장 가능(공개 편집)**
- 관리자(기존 `$_SESSION['sanctuary_admin']` 세션): 레시피 편집(응룡왕 계승 확정값 입력 등)

기존 앱과 동일 디자인 토큰(다크 + 골드) 재사용.

## 7. 기술 스택

순수 PHP + PDO/MySQL, 빌드툴 없음. `craft.php` 진입점 + 계산/액션 로직은
`craft/` 하위 파일로 분리(계산기, 시세 저장 액션, 레시피 편집 액션). 완성 후 서버 배포+테스트.

- 저티어(루비→장인→달인) 제작반 전개
- 응룡왕 외 다른 목표 티어 계산

- 로그인/회원 기능 (공개 편집)
